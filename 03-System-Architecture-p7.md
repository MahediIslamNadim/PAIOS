# Learning Intelligence Engine (LIE)

## Purpose

The Learning Intelligence Engine (LIE) is responsible for continuously optimizing the user's learning process.

It combines evidence from learning activities, projects, assessments, revision history, and behavioral patterns to recommend effective next actions.

The engine provides recommendations.

The user remains responsible for final decisions.

# Knowledge Gap Detection

The engine identifies concepts requiring additional attention.

Possible Signals

Low Quiz Accuracy

Long Time Since Review

Missing Prerequisites

Repeated Mistakes

Low Confidence

Unfinished Exercises

Project Difficulties

User Feedback

Each detected gap includes supporting evidence.

# Adaptive Study Planning

The planner creates personalized study schedules.

Factors

Available Time

Upcoming Exams

Current Projects

Energy Estimate

Knowledge Gaps

Revision Queue

Learning Goals

Difficulty Balance

Recent Progress

The schedule should remain flexible.

Inputs

Evidence Graph

Project Usage

Retention Estimate

Confidence

# Forgetting Curve Estimation

The engine estimates retention using multiple indicators.

Examples

Time Since Study

Revision Count

Quiz Accuracy

Real Project Usage

Teaching Activity

Problem Solving

Confidence

These estimates are probabilistic and should be updated as new evidence arrives.

# Cognitive Load Estimation

The engine estimates workload using observable signals.

Examples

Number of Active Subjects

Difficulty Mix

Session Duration

Task Switching

Revision Queue Size

Project Commitments

Recent Fatigue Indicators

When workload becomes excessive, recommendations should shift toward recovery and consolidation.

# Focus Intelligence

Focus recommendations are based on measurable observations.

Potential Inputs

Study Session Continuity

Keyboard Activity

Mouse Activity

Window Switching

Optional Webcam Analysis

Optional Voice Activity

Break History

The engine should avoid drawing strong conclusions from limited evidence.

---

# Explainability

Every recommendation should answer

Why this?

Why now?

What evidence supports it?

What benefit is expected?

The user should understand the reasoning behind important suggestions.

---

# Engine Constraints

The Learning Intelligence Engine

Must not replace learning.

Must not fabricate certainty.

Must not hide uncertainty.

Must not optimize only for study hours.

It should optimize for understanding, retention, and application.

---

# Success Metrics

Learning Retention

Knowledge Growth

Project Completion

Revision Health

Consistency

Reduced Knowledge Gaps

Long-Term Skill Development

---

# Summary

The Learning Intelligence Engine transforms learning activities into evidence-based recommendations.

Its purpose is not to maximize time spent studying.

Its purpose is to maximize meaningful learning.
