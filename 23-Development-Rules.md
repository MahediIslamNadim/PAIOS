# Development Rules

## Git Workflow

Branch: feature/xxx, fix/xxx, chore/xxx

Commit: Conventional Commits (feat:, fix:, chore:, docs:, test:)

PR: Template with description, screenshots, testing notes

## Code Style

TypeScript strict mode

ESLint + Prettier

Naming: camelCase (variables), PascalCase (types, classes), kebab-case (files)

## Review Process

Every PR requires 1 approval

AI review + human review

Automated checks: lint, test, build

## Documentation

Every module has README

Every API endpoint has Swagger docs

Architecture decisions recorded as ADRs
